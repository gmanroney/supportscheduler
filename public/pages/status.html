Release 0.1 05/12/2017

What has been delivered in this release

  • 10 engineers setup (via API)
  • List of engineers availavle in frontend
  • List of generated schedule available in frontend
  • Can generate schedule for future period in frontend
  • Can view schedule for future period in frontend
  • Can view schedule for particular engineer in frontend

Known Issues

Defects or functionality that needs to be fixed.

  • No useful messages returned to user when they click on retrieve or generate schedule for a period of time; makes application difficult to use
  • Problems encountered when testing of the application.
  • Use tool like Swagger to generate API documentation
  • Move RESTful calls to service using ng-requests
  • Possible for engineers to have slot assigned on two subsequent days. Function proximitycheck() to be completed and integrated into next release. Need to make sure it does not go into endless loop.
  • Lack of pagination, sorting and searching of engineers and schedules makes application difficult to use

Roadmap

Additional features not discussed which may be useful to add at future point in time.

  • Creation of engineers
  • Modification of existing engineer details
  • Accomodating more than 10 engineers (maximum TBD)
  • Accomodating less than 10 engineers (minimum of 1)
  • Login page (authentication and authorization)
  • Audit and redesign of logging to help with diagnosis of issues
  • Audit logging of actions by individual users
  • Capture e-mail address of engineers
  • Send reminders to engineers
  • Allow engineers to swap days worked with each other without supervisor involvement
